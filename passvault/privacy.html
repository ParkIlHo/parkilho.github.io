1. 개요

비번금고(이하 "본 앱")는 사용자의 비밀번호, 보안카드, 은행 계좌, 결제 카드 정보 등을 안전하게 관리하기 위한 모바일 애플리케이션입니다. 본 방침은 본 앱이 사용자의 개인정보를 어떻게 처리하는지에 대한 안내입니다.

핵심 원칙: 본 앱은 사용자의 어떤 개인정보도 외부 서버로 전송·수집하지 않습니다. 모든 데이터는 사용자 본인의 기기 내부에만 AES 암호화되어 저장되며, 개발자조차 사용자의 데이터에 접근할 수 없습니다.

2. 수집하지 않는 정보

  • 회원가입 / 계정 / 이메일 / 전화번호 등 식별 정보
  • 비밀번호 · 보안카드 · 계좌번호 · 카드 정보 등 사용자가 본 앱에 저장한 모든 데이터
  • 위치 정보, 연락처, 사진 라이브러리 등 기기 내 정보

위 정보는 본 앱이 외부로 전송하거나 개발자가 수집하지 않습니다.

3. 권한 사용

본 앱은 다음 권한을 사용하며, 각 권한은 명시된 목적으로만 사용됩니다.

3.1 카메라

보안카드 OCR(광학 문자 인식) 기능에 사용합니다. 촬영된 이미지는 사용자 기기 내부에서만 처리되며 외부로 전송되지 않습니다. 인식이 끝난 임시 이미지는 처리 직후 폐기됩니다.

3.2 Face ID / Touch ID (생체 인증)

앱 잠금 해제 용도로만 사용합니다. 생체 정보는 Apple/Google이 관리하는 시스템 영역에 있으며 본 앱은 인증 결과(성공/실패)만 받습니다.

3.3 광고 식별자(IDFA, ATT)

iOS 14 이상에서는 광고 식별자(IDFA) 사용을 위해 권한 요청 다이얼로그가 표시됩니다. 권한을 거부해도 앱 기능은 동일하게 사용할 수 있으며, 광고만 비-맞춤형으로 표시됩니다.

4. 제3자 서비스

본 앱은 다음 외부 서비스를 사용합니다. 각 서비스는 자체 개인정보 처리방침을 따릅니다.

4.1 Google AdMob

  • 목적: 앱 내 광고 표시
  • 수집 정보: 광고 식별자(IDFA/AAID), 기기 정보, 광고 노출/클릭 통계
  • Google 개인정보 처리방침: policies.google.com/privacy

4.2 Firebase Crashlytics

  • 목적: 앱 충돌(크래시) 감지 및 안정성 개선
  • 수집 정보: 충돌 시점의 스택 트레이스, 기기 모델, OS 버전 (사용자 데이터 미포함)
  • Firebase 개인정보 처리방침: firebase.google.com/support/privacy

5. 데이터 보관 및 삭제

5.1 데이터 저장 위치 및 보관 기간

  • 저장 위치: 사용자 기기 내부 (외부 서버에 저장되는 사용자 데이터 없음)
  • 암호화: AES-256
  • 보관 기간: 사용자가 직접 삭제하거나 앱을 제거할 때까지

5.2 사용자가 직접 데이터를 삭제하는 방법

본 앱은 사용자 데이터를 외부 서버에 저장하지 않으므로 별도의 삭제 요청 절차가 필요하지 않습니다. 사용자는 다음 방법으로 언제든지 직접 데이터를 삭제할 수 있습니다.

  • 개별 항목 삭제: 앱 실행 → 해당 항목(비밀번호 · 보안카드 · 계좌 · 결제카드) 선택 → 우측 상단 삭제 버튼 → 확인. 선택한 데이터는 즉시 기기 저장소에서 영구 제거됩니다.
  • 전체 데이터 일괄 삭제: 기기에서 본 앱을 삭제(uninstall)하면 앱이 저장한 모든 로컬 데이터가 자동으로 함께 삭제됩니다.
  • 광고 식별자 재설정 / 추적 권한 해제: Android: 설정 → 개인정보 보호 → 광고 → 광고 ID 재설정. iOS: 설정 → 개인정보 보호 및 보안 → 추적 → 본 앱의 추적 허용 해제.

5.3 삭제되는 데이터 / 보관되는 데이터

  • 삭제되는 데이터: 비밀번호, 보안카드, 계좌, 결제카드 등 사용자가 앱에 저장한 모든 정보 (전체 삭제 시 즉시 제거, 추가 보관 기간 없음)
  • 보관되는 데이터: 없음 (외부 서버에 보관되는 사용자 데이터가 존재하지 않습니다)
  • 제3자 서비스 데이터: Google AdMob · Firebase Crashlytics가 처리하는 데이터는 각 서비스의 자체 정책에 따릅니다. 광고 식별자 재설정 또는 OS 추적 권한 해제로 제어할 수 있습니다.

5.4 추가 문의

데이터 삭제와 관련해 추가 문의가 필요하시면 본 방침 하단의 문의 이메일로 연락해 주세요.

1. Overview

PassVault ("the App") is a mobile application that helps users securely manage passwords, security cards, bank accounts, and payment cards. This policy explains how the App handles user data.

Core principle: The App does not transmit or collect any user data to any external server. Everything is AES-encrypted and stored only on your device. The developer cannot access your data.

2. Information We Do NOT Collect

  • Account, email, phone number, or any identifying information
  • Passwords, security card numbers, account numbers, card details, or anything you store in the App
  • Location, contacts, photo library, or other device data

The App neither transmits the above information externally nor allows the developer to collect it.

3. Permissions

The App uses the following permissions strictly for the stated purposes.

3.1 Camera

Used for security card OCR. Captured images are processed entirely on-device and are not transmitted externally. Temporary images are discarded immediately after processing.

3.2 Face ID / Touch ID (Biometrics)

Used only to unlock the App. Biometric data is managed by the operating system (Apple / Google); the App only receives a success / failure result.

3.3 Advertising Identifier (IDFA, ATT)

On iOS 14+, a permission dialog requests access to the IDFA for advertising. Denying it does not affect functionality; ads will simply be shown as non-personalized.

4. Third-Party Services

The App uses the following external services, each under its own privacy policy.

4.1 Google AdMob

  • Purpose: in-app advertising
  • Collected: advertising identifier (IDFA / AAID), device info, ad impression / click stats
  • Policy: policies.google.com/privacy

4.2 Firebase Crashlytics

  • Purpose: detect crashes and improve stability
  • Collected: crash stack traces, device model, OS version (no user data)
  • Policy: firebase.google.com/support/privacy

5. Data Retention & Deletion

5.1 Where Data Is Stored and How Long

  • Storage: on-device only (no user data stored on external servers)
  • Encryption: AES-256
  • Retention: until you delete the data or uninstall the App

5.2 How to Delete Your Data

Because the App does not store any user data on external servers, no separate deletion request process is required. You can delete your data at any time using the following methods.

  • Delete individual items: Open the App → select the item (password · security card · account · payment card) → tap the delete button → confirm. The selected data is permanently removed from device storage immediately.
  • Delete all data at once: Uninstalling the App from your device automatically deletes all locally stored data.
  • Reset advertising identifier / revoke tracking: Android: Settings → Privacy → Ads → Reset advertising ID. iOS: Settings → Privacy & Security → Tracking → revoke tracking permission for this App.

5.3 Data Deleted vs. Retained

  • Deleted: all information stored in the App (passwords, security cards, accounts, payment cards). Removed immediately on deletion, with no additional retention period.
  • Retained: none (the App retains no user data on external servers).
  • Third-party data: data processed by Google AdMob and Firebase Crashlytics is governed by their own policies. You can control it by resetting your advertising identifier or revoking OS-level tracking permissions.

5.4 Further Inquiry

If you have additional questions about data deletion, please contact us using the email address at the bottom of this policy.

6. Children's Privacy

The App is not directed to children under 14, and we do not knowingly collect data from children. Because the App does not collect any user data at all, the same protection applies to every user.
